Terminal vs IDE: Why CLI is Better for Programming with AI

In this discussion, Peter Steinberger shares his journey transitioning from traditional IDE-based development to a terminal-centric workflow enhanced by AI tools. He recounts starting with cloud code in April, initially finding it lacking but appreciating the fresh approach. After experimenting with Cursor‚Äîenjoying its capabilities but frustrated by difficulties managing multiple versions‚Äîhe returned to Cloud Code as his primary tool, eventually maintaining seven subscriptions simultaneously to support his multi-window workflow.

Key Topics Discussed

Peter emphasizes that he's moved almost entirely to CLI and terminal-based workflows, using an IDE "very rarely, mostly as a diff viewer." He challenges the conventional wisdom that programmers need to read every line of code, arguing that most software fundamentally does the same thing: transforms data from one shape to another. He notes he doesn't find reading code exciting, especially mundane aspects like Tailwind button alignments or routine data processing pipelines.

Main Takeaways

  • Terminal-first approach: Peter runs multiple windows side by side entirely through CLI, finding it more efficient than traditional IDEs for AI-assisted work
  • Minimal IDE usage: He uses his IDE almost exclusively as a diff viewer rather than for active development
  • Code as data transformation: Most programming is simply moving and reshaping data‚Äîa perspective that reduces the need to read every line
  • AI changes the game: The combination of AI tools and terminal workflows has made traditional IDE features less necessary
  • Selective reading: Not all code needs to be read; programmers can focus on high-level logic rather than implementation details
